# Tempo 后续优化与完善计划

本文只记录当前版本之后仍需要继续设计和实现的内容。已完成的功能不再作为待办展开，包括：基础番茄钟、Today/Review/Settings、本地数据保存、无 GPU 启动兼容、Windows 便携包与应用图标、Today Tasks、当前任务、任务编辑、任务排序、Inbox 持久化、Inbox 转任务、Focus-only Session Notes、End Day 科学复盘模板、未完成任务顺延到明天、Review 周复盘展示、Review 历史 End Day 编辑、应用内专注大屏和 Windows 真全屏 Focus。

## 1. 产品方向

Tempo 不是重型项目管理软件，而是一个围绕“计划、专注、记录、复盘”的轻量桌面工作台。

后续设计应继续遵守：

- Today 优先回答“现在做什么”和“今天还剩什么”。
- Focus 状态尽量减少干扰，只保留当前任务、倒计时、必要控制和 Inbox。
- Review 承担复盘与洞察，不把 Today 做成复杂报表。
- Inbox 保持轻量，适合捕捉随机想法，不自动按日期归档。
- 数据继续优先本地保存，导入导出要稳定可靠。

## 2. Review 深度分析

当前 Review 已经能查看日历、周 End Day、Session Notes、Day Thread 和 Focus Herbarium。后续重点是提高复盘的信息密度。

### 2.1 按任务聚合 Session Notes

把 Review 中的 focus sessions 从“按时间列表”升级为“按任务聚合”：

- 每个任务下列出相关 focus sessions。
- 显示该任务总投入分钟数。
- 显示该任务所有 session notes。
- 未绑定任务的 session 放入 `Unassigned Focus`。
- 支持按日期或本周范围查看任务投入。

### 2.2 周复盘 Summary

在现有 Weekly End Day 列表上方增加周统计摘要：

- 本周专注总分钟数。
- 本周完成任务数。
- 本周中断次数。
- 本周最常见的 blocker。
- 下周建议优先处理的任务或主题。

可选增强：导出本周复盘为 Markdown。

## 3. Day Thread 增强

当前 Day Thread 展示已经保存的 session。后续可以显示正在进行中的 focus：

- Focus 运行时，在 Day Thread 上显示一段实时增长的 bar。
- 暂停时保留该 bar，但弱化显示。
- 中断时保存为 interrupted session。
- hover 显示开始时间、预计结束时间、当前任务和 note。

技术要点：

- 需要把 timer active segment 抽到 App 层或独立 store。
- Day Thread 合并渲染已保存 session 和当前 active segment。
- 继续保证切换 Today/Review 不会丢失倒计时。

## 4. Inbox 组织能力

Inbox 已经支持新增、编辑、删除、持久化和转为今日/明日任务。后续可以增强组织能力，但仍保持轻量：

- Inbox item 手动排序。
- 一键清空已处理 item。
- 给 Inbox item 增加轻量标签。
- 导出 Inbox 为 Markdown。

不建议做：

- 不做自动日期归档。
- 不做复杂分类树。
- 不做多层嵌套笔记。

## 5. 设置与预设模式

设置页可以加入专注预设：

- Classic: 25 / 5 / 15 / 4
- Deep Work: 50 / 10 / 30 / 3
- Writing: 40 / 8 / 20 / 4
- Reading: 30 / 5 / 15 / 4

交互要求：

- 点击预设后覆盖当前 timer 设置。
- 用户仍然可以手动修改。
- 可保存自定义预设。
- 自定义预设保存到本地配置。

## 6. 发布与安装体验

当前目标是保持 GitHub Release 的 Windows 下载体验稳定。后续继续完善：

- 稳定生成 `Tempo Setup x.y.z.exe`。
- 安装器使用自定义 icon。
- 桌面快捷方式和开始菜单快捷方式稳定生成。
- README 解释 Windows SmartScreen 提示。
- 后续考虑代码签名证书。
- 后续考虑自动更新机制。

## 7. 数据与迁移

当前本地数据包括：

- `preferences.json`
- `sessions.json`
- `tasks.json`
- `dayJournals.json`
- `inbox.json`

后续修改数据结构时需要：

- 保持旧数据可读取。
- 为新增字段设置默认值。
- 导入导出包含所有本地数据。
- 不轻易提高 schema version，除非确实需要显式迁移逻辑。

## 8. iPhone 迁移准备

暂不开发移动端，但数据结构应保持可迁移：

- Today 未来可变成移动端一个 tab。
- Review 可改成纵向月份列表。
- Day Thread 可改成纵向 timeline。
- Task、Session、DayJournal、InboxItem 数据结构保持稳定。
- 本地导入导出先行，云同步后置。

## 9. 建议优先级

下一阶段建议按这个顺序继续：

1. Review 按任务聚合 session 和 note。
2. 周复盘 Summary 与 Markdown 导出。
3. Day Thread 显示当前正在进行的 session。
4. Inbox 排序、标签和 Markdown 导出。
5. 预设模式和自定义预设。
6. 正式 Windows 安装器稳定性、签名与自动更新。
